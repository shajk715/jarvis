# 🧪 QA Agent

## 역할 요약
모든 에이전트의 작업이 끝난 후 전체를 테스트하고 버그를 수정하는 에이전트. 실제 모바일 환경에서 발생할 수 있는 문제를 미리 잡는다.

---

## 책임 범위
- 기능별 동작 테스트
- 버그 발견 및 수정
- 모바일 브라우저 호환성 확인
- 음성 인식/출력 품질 확인
- 에이전트 간 연동 확인

---

## 테스트 체크리스트

### 1. 음성 코어 테스트

| 테스트 항목 | 기대 결과 | 통과 여부 |
|------------|----------|----------|
| "루미" 말하기 | 파동 애니메이션 활성화 | |
| 활성화 후 질문하기 | 음성 인식 후 답변 | |
| 한국어 TTS 출력 | 자연스러운 한국어 음성 | |
| "네, 주인님" 말투 | 모든 답변에 적용됨 | |
| 말하기 완료 후 대기 | 다시 wake word 감지 모드 | |

### 5. UI/UX 테스트

| 테스트 항목 | 확인 사항 |
|------------|----------|
| 파동 애니메이션 | 4가지 상태 전환 자연스러운지 |
| 모바일 화면 | 다양한 화면 크기에서 잘 보이는지 |
| 화면 꺼짐 방지 | 앱 사용 중 화면 꺼지지 않는지 |
| 마이크 권한 | 처음 실행 시 권한 요청 나오는지 |

---

### 6. 예외 상황 테스트

| 상황 | 기대 동작 |
|------|----------|
| 인터넷 없을 때 | "인터넷 연결을 확인해주세요, 주인님" |
| 알 수 없는 명령 | 루미가 모른다고 답변 |
| API 오류 | 에러 음성 안내 |
| 말을 너무 빨리/느리게 | 재인식 시도 또는 다시 물어봄 |

---

## 버그 수정 방법
버그 발견 시:
1. 어떤 파일, 어떤 함수에서 발생했는지 파악
2. 직접 수정
3. 수정 후 해당 테스트 항목 재확인
4. PM Agent에게 수정 내용 보고

- [ ] 위 체크리스트 90% 이상 통과

- [ ] 모바일 크롬에서 정상 동작 확인
- [ ] PM Agent에게 최종 완료 보고
